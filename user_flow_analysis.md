# Localite App 用戶使用流程分析

## 📋 專案概述

Localite 是一款結合 AI 導覽員的地點導覽應用，專為博物館、美術館等場域設計，提供沉浸式語音導覽體驗。應用支援一般遊客和商戶兩種角色，提供即時語音導覽、互動問答、照片分析、旅程記錄生成等功能。

## 🎯 主要用戶角色

1. **一般用戶（遊客）** - 使用 AI 導覽功能探索場域，獲得個人化的導覽體驗
2. **商戶用戶** - 管理自己的場域和導覽內容，提供專業的導覽服務

## ⚠️ 發現的流程問題

### 問題 1: 認證流程與訪客體驗的平衡

```
❌ 問題點:
• 訪客模式功能受限過多，影響初次使用的體驗
• 製作旅程紀錄時的登入檢查造成流程中斷
• 個人檔案頁與主要導覽流程的整合不夠順暢
```

### 問題 2: 場域進入方式的多樣性與一致性

```
❌ 問題點:
• GPS 地圖與 QR 掃描的用戶體驗差異過大
• 兩種進入方式的轉場和資訊同步需要優化
• 景點介紹頁面需要適應不同的進入來源
```

### 問題 3: AI 導覽體驗的完整性

```
❌ 問題點:
• 文字對話與 TTS 播報的同步機制需要完善
• 多媒體卡片的載入速度和順序控制
• 拍照指引與問答環節的互動體驗優化
```

### 問題 4: 旅程紀錄製作的複雜度

```
❌ 問題點:
• 照片上傳流程過於繁複，可能造成用戶流失
• EPSON 列印功能的整合與錯誤處理
• 登入狀態檢查在關鍵節點可能造成挫折
```

### 問題 5: 徽章系統與持續使用的動機

```
❌ 問題點:
• 徽章獲得時機與用戶預期的落差
• 徽章系統的視覺設計與解鎖機制
• 「再探索」決策點的用戶引導不夠明確
```

### 問題 6: 商戶管理流程的專業性

```
❌ 問題點:
• 場域管理介面對於非技術人員的友好度
• 統計數據的可視化與實用性
• 商戶認證與權限管理的完整性
```
